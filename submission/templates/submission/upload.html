Supported upload formats: zip, tar

Maximum allowed upload file size:

The following are requirements concerning the upload of data packages:
  1. The container must have the file extension tar, tar.gz, or zip
  2. The container file (tar, tar.gz, zip) file must contain a single root folder (it is not allowed to have more than one folder at the root level)
  3. The container file name must be a safe name*
  4. The package name being the name without the extension (.tar, .tar.gz, .zip), the root folder of the container must be the same as the package name
  5. The package name must not be one of representations, schemas, metadata, documentation (reserved words)
  6. The root folder of the container needs to contain the folder "metadata"
  7. If the container has content data, these must be stored in the "representations" folder followed by a safe name* representation label and a data folder, e.g. "representations/rep1/data/image.jpeg"

*Safe names: Only ascii characters, numbers, dashes, underscores, and dots are allowed in safe names

The following is an example of a minimal structure of a data package:

mypackage/
├── metadata
│   └── dc.xml
└── representations
    └── rep1
        └── data
            └── myfile.pdf

Additionally, the folders "schemas" and "documentation" can be added at the root or representation level (refer to the Common Specification of Information Packages (CSIP) for further information).
